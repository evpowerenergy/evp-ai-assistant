# การวิเคราะห์: ทำไม "เดือนที่แล้วมีลูกค้าใหม่กี่ราย" ถึงเลือก Tool ผิด

## ปัญหาที่เกิดขึ้น

| คำถาม | ผลลัพธ์ | Tool ที่น่าจะถูกเรียก |
|-------|----------|------------------------|
| **เดือนที่แล้ว มีลูกค้าใหม่กี่ราย** | "ไม่มีข้อมูลในระบบที่ตอบคำถามนี้ได้" (อ้าง Sales Closed, ไม่มีฟิลด์ลูกค้าใหม่) | ควรใช้ **search_leads** (นับลีดที่สร้างในเดือนนั้น) |
| **เดือนที่แล้วมีลีดใหม่กี่ราย** | พบ 614 leads \| ช่วง 2026-01-01 - 2026-01-31 | **search_leads** ถูกเรียกถูกต้อง |

ทั้งสองคำถามในทางธุรกิจหมายถึง "จำนวนรายที่เข้ามาใหม่ในเดือนนั้น" (นับจากช่วงวันที่สร้าง) แต่คำถามแรกถูกตีความและเลือก tool ผิด

---

## สาเหตุหลัก

### 1. ลำดับและความกว้างของ Keyword ใน Fallback (`llm_router.py`)

ใน `_suggest_default_tool_for_data_request()` ตรวจคำหลักตามลำดับ:

1. `DATA_KEYWORDS_SALES_UNSUCCESSFUL` (ปิดไม่ได้, ปิดไม่สำเร็จ ฯลฯ)
2. **`DATA_KEYWORDS_SALES_CLOSED`** ← มีคำว่า **"เดือนที่แล้ว"** อยู่ด้วย
3. อื่นๆ (appointments, team_kpi, …)
4. `DATA_KEYWORDS_LEADS` (ลีด, lead, **ลูกค้า**, …)

ข้อความ **"เดือนที่แล้ว มีลูกค้าใหม่กี่ราย"**:
- ไม่ตรง SALES_UNSUCCESSFUL
- **ตรง SALES_CLOSED เพราะมี "เดือนที่แล้ว"** → fallback ส่งกลับ **get_sales_closed**
- จึงไม่เคยไปถึง LEADS → ไม่ได้ suggest **search_leads**

ดังนั้นถ้า LLM คืน `general` หรือไม่เรียก tool ระบบจะ inject **get_sales_closed** ให้ ทำให้คำถาม "ลูกค้าใหม่" ไปใช้ tool ของยอดปิดการขาย

### 2. ความหมาย "ลูกค้าใหม่" ไม่ได้ถูกนิยามใน Prompt

ใน system prompt และ tool descriptions:

- มีตัวอย่างชัดสำหรับ "ลีดวันนี้", "เดือนที่แล้วปิดการขาย**ได้**กี่ราย", "ปิดการขาย**ไม่ได้**กี่ราย"
- **ไม่มีตัวอย่างหรือกฎสำหรับ "ลูกค้าใหม่กี่ราย" / "มีลูกค้าใหม่กี่ราย"**

ใน CRM บริบทนี้ "ลูกค้าใหม่" มักใช้ในความหมาย:
- **ความหมายที่ใช้บ่อย:** ลีด/คนที่เข้ามาใหม่ในระบบ = นับจาก **วันที่สร้างลีด (lead created_at)** → ควรใช้ **search_leads** + date_from/date_to (ไม่ใส่ status) แล้วนับจำนวนลีด = เทียบได้กับ "ลูกค้าใหม่" ในความหมายนี้

แต่เพราะไม่มีกฎนี้ใน prompt:
- LLM อาจตีความ "ลูกค้า" = คนที่ซื้อแล้ว → โน้มไป **get_sales_closed**
- หรือถ้าใช้ fallback อยู่แล้ว (ข้อ 1) ก็จะได้ get_sales_closed อยู่ดี

### 3. ทำไม "เดือนที่แล้วมีลีดใหม่กี่ราย" ถึงถูก

- คำว่า **"ลีด"** ชัดเจน → LLM มักเลือก **search_leads** พร้อม date_from/date_to ถูกต้อง
- แม้ fallback จะมี "เดือนที่แล้ว" อยู่ใน SALES_CLOSED แต่ถ้า LLM เองเลือก tool ได้ (db_query + search_leads) ระบบก็จะไม่ใช้ fallback และได้ผล 614 leads ตามที่เห็น

ความต่างคือคำถามแรกไม่มีคำว่า "ลีด" มีแต่ "ลูกค้าใหม่" จึงทั้ง fallback และการตีความของ LLM โน้มไปทาง get_sales_closed

### 4. ทำไมคำตอบถึงเป็น "ไม่มีข้อมูลในระบบที่ตอบคำถามนี้ได้"

หลังจากเรียก **get_sales_closed**:
- ผลลัพธ์เป็น salesCount (จำนวน QT), totalSalesValue, salesLeads (รายการปิดการขาย)
- ไม่มีฟิลด์ชื่อ "จำนวนลูกค้าใหม่" หรือ "unique new customers" โดยตรง
- Response generator จึงสรุปว่าข้อมูลที่มีไม่ตรงกับคำถาม "จำนวนลูกค้าใหม่เดือนที่แล้ว" และตอบว่าไม่มีข้อมูลในระบบที่ตอบคำถามนี้ได้

---

## สรุปสาเหตุ

| ปัจจัย | ผลต่อคำถาม "ลูกค้าใหม่กี่ราย" | ผลต่อคำถาม "ลีดใหม่กี่ราย" |
|--------|--------------------------------|-----------------------------|
| Fallback keyword "เดือนที่แล้ว" อยู่ใน SALES_CLOSED | ไป get_sales_closed (ผิด) | ถ้า LLM เลือก search_leads เอง จะไม่ใช้ fallback (ถูก) |
| ไม่มีกฎใน prompt ว่า "ลูกค้าใหม่" = นับลีดที่สร้างในช่วง | LLM โน้มไป get_sales_closed หรือไม่แน่ใจ | มี "ลีด" ชัด → โน้มไป search_leads |
| ความหมาย "ลูกค้า" ใน prompt = คนที่ซื้อแล้ว | เสริมให้เลือก get_sales_closed | ไม่กระทบ |

---

## แนวทางแก้ไข (สรุป)

1. **ลดการ match ผิดของ fallback**
   - อย่าให้คำว่า "เดือนที่แล้ว" อย่างเดียวไป trigger get_sales_closed ได้
   - เช่น เอา "เดือนที่แล้ว" ออกจาก `DATA_KEYWORDS_SALES_CLOSED` หรือใช้เฉพาะวลีเช่น "เดือนที่แล้วปิดได้", "ปิดการขายได้กี่ราย"
   - หรือเพิ่มกฎ: ถ้าข้อความมี "ลูกค้าใหม่" หรือ "ลีดใหม่" และไม่มี "ปิดการขาย" → ให้ fallback ไป **search_leads** แทน

2. **เพิ่มกฎและตัวอย่างใน system prompt / intent**
   - บอกชัดว่า: **"ลูกค้าใหม่กี่ราย" / "มีลูกค้าใหม่กี่ราย" ในช่วงเวลา** = นับลีดที่**สร้าง**ในช่วงนั้น (lead created in period) → ใช้ **search_leads** กับ date_from/date_to (ไม่ใส่ status)
   - ตัวอย่าง: "เดือนที่แล้วมีลูกค้าใหม่กี่ราย" → db_query, ใช้ search_leads, date_from/date_to = เดือนที่แล้ว

3. **(ถ้าต้องการแยกระดับความหมาย)** ใน prompt อธิบายว่า:
   - "ลูกค้าใหม่" / "ลีดใหม่" ในช่วง = ใช้ search_leads (นับจาก created_at)
   - "ลูกค้าที่ปิดการขายใหม่ในเดือนนั้น" ถ้าถามชัดแบบนี้ = ใช้ get_sales_closed และนับ unique lead จาก salesLeads

เมื่อแก้ทั้ง fallback และ prompt แล้ว คำถาม "เดือนที่แล้วมีลูกค้าใหม่กี่ราย" ควรได้ tool เป็น **search_leads** และได้ตัวเลขในลักษณะเดียวกับ "ลีดใหม่กี่ราย" (นับจากช่วงวันที่สร้างลีด)
